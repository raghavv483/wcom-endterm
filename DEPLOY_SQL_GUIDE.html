🚀 WaveLearn - Database Setup

Complete deployment of quiz system tables and policies

Step 1: Access Supabase Dashboard

Go to Supabase Dashboard and open the SQL Editor:

Open Supabase Dashboard
  1. Select your wcom-endterm project
  2. Click SQL Editor in the left sidebar
  3. Click "New Query" button

Step 2: Create Quiz Tables

Copy and paste the following SQL into your Supabase SQL Editor:

-- Quiz System Tables -- Create users table if it doesn't exist CREATE TABLE IF NOT EXISTS users ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), email VARCHAR(255) UNIQUE NOT NULL, username VARCHAR(100), password_hash VARCHAR(255), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, last_login TIMESTAMP, role TEXT DEFAULT 'user' ); -- Create quizzes table CREATE TABLE IF NOT EXISTS quizzes ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), admin_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, title TEXT NOT NULL, description TEXT, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Create quiz_questions table CREATE TABLE IF NOT EXISTS quiz_questions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), quiz_id UUID NOT NULL REFERENCES quizzes(id) ON DELETE CASCADE, question TEXT NOT NULL, options TEXT[] NOT NULL, correct_index INT NOT NULL CHECK (correct_index >= 0 AND correct_index < 4), order_index INT DEFAULT 0, created_at TIMESTAMPTZ DEFAULT NOW() ); -- Create quiz_attempts table CREATE TABLE IF NOT EXISTS quiz_attempts ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), quiz_id UUID NOT NULL REFERENCES quizzes(id) ON DELETE CASCADE, user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, score INT, total_questions INT, attempted_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(quiz_id, user_id) ); -- Create follows table CREATE TABLE IF NOT EXISTS follows ( follower_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, following_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT NOW(), PRIMARY KEY (follower_id, following_id), CHECK (follower_id != following_id) ); -- Create indexes CREATE INDEX IF NOT EXISTS idx_quizzes_admin_id ON quizzes(admin_id); CREATE INDEX IF NOT EXISTS idx_quizzes_created_at ON quizzes(created_at); CREATE INDEX IF NOT EXISTS idx_quiz_questions_quiz_id ON quiz_questions(quiz_id); CREATE INDEX IF NOT EXISTS idx_quiz_attempts_quiz_id ON quiz_attempts(quiz_id); CREATE INDEX IF NOT EXISTS idx_quiz_attempts_user_id ON quiz_attempts(user_id); CREATE INDEX IF NOT EXISTS idx_follows_follower_id ON follows(follower_id); CREATE INDEX IF NOT EXISTS idx_follows_following_id ON follows(following_id); CREATE INDEX IF NOT EXISTS idx_users_role ON users(role);

Step 3: Disable RLS Policies

After the first query completes, create a new query and paste this SQL to disable RLS:

-- Disable RLS on all quiz tables ALTER TABLE quizzes DISABLE ROW LEVEL SECURITY; ALTER TABLE quiz_questions DISABLE ROW LEVEL SECURITY; ALTER TABLE quiz_attempts DISABLE ROW LEVEL SECURITY; ALTER TABLE follows DISABLE ROW LEVEL SECURITY;

Step 4: Execute and Verify

  1. In the SQL Editor, click "Run" or press Ctrl+Enter
  2. Wait for the query to complete (you should see "✓ All done")
  3. Repeat for both SQL blocks
  4. Check the Results tab to confirm no errors
✅ After deployment:
  1. Refresh your browser (F5) at localhost:5174
  2. Go to Admin Dashboard in the navigation
  3. Click "Create New Quiz"
  4. Start creating quizzes! 🎉
⚠️ Troubleshooting:
  • If you see "CREATE TABLE IF NOT EXISTS" errors with duplicate key violations, that's OK - the tables might already exist
  • Make sure both SQL blocks execute without errors in the final Results section
  • If RLS disable fails, the tables might not have RLS enabled - that's fine too
